# Domain Docs

This repo uses a single-context domain layout.

Before exploring code or drafting issues, read `CONTEXT.md` at the repo root for domain language. Read relevant ADRs under `docs/adr/` for accepted architectural decisions.

Use glossary terms exactly when naming domain concepts in issue titles, test names, implementation notes, and refactor proposals. If an output would contradict an ADR, call that out explicitly instead of silently overriding it.
